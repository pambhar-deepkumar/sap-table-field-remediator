Snippet 1  FI Line Items — Direct BSEG Read    [ COMPATIBILITY VIEW ]
Custom report reading FI line items directly from BSEG. In S/4HANA, BSEG is a compatibility view over ACDOCA. Direct access works but key fields are renamed.

ECC CODEDATA: lt_bseg TYPE TABLE OF bseg,
      ls_bseg TYPE bseg.
 
SELECT * FROM bseg
  INTO TABLE lt_bseg
  WHERE bukrs = '1000'
    AND gjahr = '2023'
    AND belnr = '0100000123'.
 
LOOP AT lt_bseg INTO ls_bseg.
  WRITE: / ls_bseg-hkont, ls_bseg-dmbtr, ls_bseg-buzei.
ENDLOOP.

What to fix:  Replace BSEG with ACDOCA. Map HKONT → RACCT, BUZEI → DOCLN, BUKRS → RBUKRS. Filter with RLDNR = '0L' for leading ledger.

Snippet 2  Pricing Conditions — Direct KONV Read    [ ABOLISHED ]

Custom program reading pricing conditions from KONV (a cluster table). KONV is abolished in S/4HANA. The replacement is PRCD_ELEMENTS (available from S/4HANA 1909 onwards).

ECC CODEDATA: lt_konv TYPE TABLE OF konv,
      ls_konv TYPE konv.
 
SELECT * FROM konv
  INTO TABLE lt_konv
  WHERE knumv = lv_knumv
    AND kposn = '000010'.
 
LOOP AT lt_konv INTO ls_konv.
  WRITE: / ls_konv-kschl, ls_konv-kbetr, ls_konv-waers.
ENDLOOP.

What to fix:  Replace KONV with PRCD_ELEMENTS. Verify S/4HANA release is 1909 or later. Field names are the same in PRCD_ELEMENTS.


Snippet 3  Change Documents — Direct CDPOS Read    [ ABOLISHED ]

Program reading change document items from CDPOS (a cluster table). CDPOS cluster is abolished. Data is now in transparent tables CDPOS_STR and CDPOS_UID.

ECC CODEDATA: lt_cdpos TYPE TABLE OF cdpos,
      ls_cdpos TYPE cdpos.
 
SELECT * FROM cdpos
  INTO TABLE lt_cdpos
  WHERE objectid = lv_objectid
    AND tabname  = 'VBAK'.
 
LOOP AT lt_cdpos INTO ls_cdpos.
  WRITE: / ls_cdpos-fname, ls_cdpos-value_new, ls_cdpos-value_old.
ENDLOOP.

What to fix:  Replace CDPOS with CDPOS_STR for string values or CDPOS_UID for ID-based values. Same fields available in transparent table.

Snippet 6  Material Document — MSEG with Renamed Fields    [ RESTRUCTURED ]

Goods movement report joining MSEG and MKPF. MSEG still exists but some fields changed. Verify all field names against SE11 on target system.

ECC CODEDATA: lt_mseg TYPE TABLE OF mseg,
      ls_mseg TYPE mseg.
 
SELECT m~mblnr m~zeile m~matnr m~werks
       m~menge m~meins m~bwart
  FROM mseg AS m
  INNER JOIN mkpf AS k ON k~mblnr = m~mblnr
                      AND k~mjahr = m~mjahr
  INTO TABLE lt_mseg
  WHERE m~werks = '1000'
    AND m~bwart IN ('101', '261').

What to fix:  MSEG and MKPF still exist but verify field positions via SE11. Test on S/4HANA system and compare field list. Add MANDT explicitly if needed.

Snippet 9  FI — BSEG with HKONT and BUZEI    [ COMPATIBILITY VIEW ]

GL account balance report using field HKONT (renamed to RACCT in ACDOCA) and BUZEI (renamed to DOCLN). Common source of wrong data if not fixed.

ECC CODEDATA: ls_bseg TYPE bseg.
 
SELECT SINGLE hkont buzei dmbtr shkzg
  FROM bseg
  INTO (ls_bseg-hkont, ls_bseg-buzei,
        ls_bseg-dmbtr, ls_bseg-shkzg)
  WHERE bukrs = lv_bukrs
    AND belnr = lv_belnr
    AND gjahr = lv_gjahr
    AND buzei = lv_buzei.

What to fix:  Rewrite against ACDOCA. Map: BUKRS→RBUKRS, HKONT→RACCT, BUZEI→DOCLN. Add RLDNR = '0L' for leading ledger. Use SHKZG equivalent HSL sign logic.


Snippet 10  Pricing — KONV Condition Type Loop    [ ABOLISHED ]

SD pricing logic looping through all condition types for a document from KONV. Full KONV table access — cluster will dump in S/4HANA.

ECC CODEDATA: lt_konv TYPE TABLE OF konv,
      ls_konv TYPE konv.
 
SELECT knumv kposn kschl kwert kbetr waers
  FROM konv
  INTO TABLE lt_konv
  WHERE knumv = lv_knumv.
 
LOOP AT lt_konv INTO ls_konv
  WHERE kschl = 'PR00' OR kschl = 'MWST'.
  lv_total = lv_total + ls_konv-kwert.
ENDLOOP.

What to fix:  Replace KONV with PRCD_ELEMENTS. Same field names. Ensure S/4HANA release is 1909+. Loop logic can remain unchanged after table name is corrected.


Snippet 11  Change Documents — CDHDR and CDPOS Join    [ ABOLISHED ]

Audit trail program joining CDHDR and CDPOS to track changes to sales orders. CDPOS cluster abolished — CDHDR still exists but join target must change.

ECC CODESELECT h~changenr h~objectid h~udate h~utime
         p~tabname p~fname p~value_new p~value_old
  FROM cdhdr AS h
  INNER JOIN cdpos AS p ON p~changenr = h~changenr
  INTO TABLE @DATA(lt_changes)
  WHERE h~objectclas = 'VERKBELEG'
    AND h~objectid   = lv_vbeln.

What to fix:  Replace CDPOS with CDPOS_STR in the JOIN. CDHDR remains. Test join on target system. CDPOS_STR contains the same fields for string-based change documents.

Snippet 12  FI — Universal Journal via Old BSEG Pattern    [ COMPATIBILITY VIEW ]

Period-end GL balance report summing amounts from BSEG by cost centre and GL account. Should be rewritten to use ACDOCA for accuracy and performance.

ECC CODESELECT hkont kostl SUM( dmbtr ) AS total_amount
  FROM bseg
  INTO TABLE @DATA(lt_balances)
  WHERE bukrs = '1000'
    AND gjahr = '2023'
    AND monat IN ('01', '02', '03')
  GROUP BY hkont kostl.

What to fix:  Rewrite using ACDOCA. Map HKONT→RACCT, KOSTL→KOSTL (same), BUKRS→RBUKRS, MONAT→POPER. Add RLDNR = '0L'. Aggregation logic remains the same.
